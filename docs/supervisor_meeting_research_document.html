1. Executive Summary

The project aims to develop a vehicle-specific real-time accident risk prediction and alert system for mountainous roads, with Kadugannawa selected as the primary study area. The core idea is that cars, buses, lorries, motorcycles, vans, three-wheelers, and pick-ups do not experience the same risk under the same road and weather conditions. A steep sharp bend during rain is more hazardous for a loaded lorry or motorcycle than for a small car, so the alert system must consider vehicle type together with weather, road geometry, time, driver, and telemetry conditions.

The proposal originally positioned Random Forest as the primary machine learning model because it can model non-linear patterns, handle mixed feature types, and provide interpretable feature importance. The implementation has now expanded the modelling stage to include Random Forest tuning, SMOTE-based imbalance handling, XGBoost comparison, and SHAP explainability. This strengthens the academic contribution because it allows the final study to report both the proposed model and a stronger comparative baseline.

Current status: A complete research prototype pipeline exists. It includes mock police accident records, real weather data, OpenMeteo API integration, OpenStreetMap road geometry extraction, preprocessing, feature engineering, Random Forest and XGBoost training, model evaluation, SHAP explanation, backend API structure, frontend dashboard pages, and an edge-device telemetry simulator.

3. Research Problem and Research Questions

Problem Statement

Mountainous roads such as Kadugannawa contain steep gradients, sharp curves, sudden weather changes, and visibility limitations. Existing accident prediction systems are mainly designed for urban or highway environments, often treat all vehicles equally, and rarely combine historical accident patterns with real-time weather and terrain conditions. Therefore, there is a need for a vehicle-specific, terrain-aware machine learning model that can provide early accident risk alerts.

Research Questions

  1. What machine learning algorithms have been used for early road accident prediction?
  2. What road-safety factors in mountainous areas and vehicle-specific factors influence accident risk?
  3. How can an accurate machine learning algorithm be developed to identify accident risk in advance within mountainous environments?
  4. How can the algorithm be evaluated to ensure it supports safer driving through early warnings?

4. Research Goal and Objectives

The main objective is to develop a vehicle-specific real-time accident risk prediction and alert system for mountainous roads so that drivers can receive early warnings and make safer decisions.

7. Data Preprocessing and Integration

The preprocessing pipeline merges accident, weather, road geometry, and telemetry-related features into a model-ready dataset. A key issue handled in the implementation is date alignment: accident dates and weather dates must be normalised before merging to prevent silent join failures. The pipeline also avoids leakage by removing outcome-derived columns such as injuries, fatalities, and manually generated risk scores before modelling.

Preprocessing Area Implementation
Train-test split Held-out test set retained before oversampling to avoid synthetic sample leakage.
Class imbalance SMOTE applied to training data only. Final post-SMOTE training set in the preprocessing report: 510 rows, balanced as 170 Low, 170 Medium, 170 High.
Missing values KNN and median imputers created for model features and stored for inference reuse.
Scaling Scaler fitted only on training data; test set transformed using the fitted training scaler.
Output artifacts X_train.csv, X_test.csv, y_train.csv, y_test.csv, feature_names.csv, and preprocessing report.
Important academic point: The primary metric should be F1-macro rather than accuracy because high-severity accidents are the minority class. A high accuracy score alone could hide poor High-risk detection.

11. Explainability and Research Findings

SHAP analysis was applied to the tuned Random Forest model on the held-out test set. This is useful academically because it explains why the model predicts Low, Medium, or High risk. It also gives evidence for the research claim that weather, visibility, road geometry, and vehicle type interact in mountain-road accidents.

Finding 1: Weather and visibility dominate High severity. Visibility, visibility class, and weather condition are the strongest High-risk predictors in the SHAP report.
Finding 2: Road geometry compounds risk. Slope, curvature, and the slope x curvature interaction show that steep sharp bends create non-linear danger.
Finding 3: Vehicle type supports the proposal. Vehicle risk weight, lorry-steep interaction, and motorcycle-rain interaction support vehicle-specific risk modelling.
Finding 4: Time matters. Night-time and monsoon-season flags increase risk and support real-time contextual alerting.

Beyond model training, the project has been organised as a deployable software system. The current architecture contains a backend, frontend, data processing pipeline, trained models, and an edge-device simulation layer.

Data pipeline src/data_collection, src/preprocessing, src/feature_engineering Collect weather/road data, merge records, and generate model-ready features.

Backend backend/api/predict.py, weather.py, segments.py, telemetry.py Expose prediction, weather, road segment, and telemetry endpoints.

The intended runtime flow is: driver selects or sends vehicle type, system fetches current weather, nearest road segment features and telemetry are added, the model predicts risk level, and the interface displays a Low, Medium, or High warning with vehicle-specific alert messaging.

14. Limitations and Research Risks

Risk Impact Mitigation
Mock accident data Model scores may be optimistic because the current accident records are simulated. Clearly state that current results are prototype results. Replace with official police data when available and rerun all experiments.
Class imbalance High-severity accidents are rare, so accuracy alone can be misleading. Use F1-macro, High-class recall, High-class precision, and SMOTE inside cross-validation.
Data leakage Outcome variables such as injuries or fatalities could inflate performance. Drop leakage columns before modelling and document this in the methodology.
Weather granularity Daily weather may not represent accident-time conditions accurately. Use OpenMeteo hourly or current API data for deployment and, if possible, accident-time matching.
Road geometry accuracy OSM data may not perfectly represent local curve sharpness or surface condition. Validate high-risk locations manually and compare with police/local authority knowledge.
Deployment validation Prototype may behave differently with delayed APIs or weak network conditions. Add caching, fallback weather values, offline road features, and input validation tests.
